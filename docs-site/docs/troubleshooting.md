# 故障排查手册

排查建议按三层推进：输入是否有效、解析是否完整、输出是否可执行。

## 上传后没有表

优先检查：

- 当前 Sheet 是否正确
- 结构区是否被覆盖
- 模板是否混入大量说明文本

收敛动作：

- 切换到含表定义的 Sheet
- 在电子表格视图缩小解析范围
- 用标准样例验证链路是否正常

## 表名或字段识别不完整

优先检查：

- 是否存在 `物理名` 与 `データ型` 列
- 列标题是否偏移
- 合并单元格是否破坏结构

收敛动作：

- 去掉装饰行后重试
- 回看解析摘要与字段预览

## DDL 为空或导出失败

优先检查：

- 是否至少有一张完整表
- 是否存在缺失数据类型字段
- 方言是否与目标库一致

收敛动作：

- 先修复预览问题，再重新生成
- 使用 Name Fix 收敛命名异常
- ZIP 导出后查看跳过清单

## 应用级异常保护

当运行时出现前端异常，系统会进入错误兜底界面。

可用动作包括：

- 重新加载应用
- 打开开发者模式
- 按 F12 查看调试信息

## 问题反馈建议

若仍无法定位，请在 GitHub issue 提供：

- 复现路径
- 当前截图
- 最小可复现 Excel 样例
- 预期结果与实际结果

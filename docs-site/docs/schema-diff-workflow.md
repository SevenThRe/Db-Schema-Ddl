# Schema Diff 差分工作流

这页覆盖 `Diff` 视图的完整能力：版本对比、重命名确认、ALTER 预览与导出。

## 能力边界

Schema Diff 的目标是把“新旧定义差异”转成可执行变更脚本，而不是重新生成全量建表 SQL。

- 适合增量发布、回归比对、结构评审
- 支持按文件版本选择基线并保留历史
- 支持人工确认重命名，降低误判风险

## 入口与前置条件

进入方式：

- 工作台中部预览区切换到 `Diff` 标签

前置条件：

- 至少有一个已上传文件
- 当前已选中目标文件与 Sheet

## 版本选择与历史

Schema Diff 会基于当前文件，自动加载可用历史版本作为候选基线。

- 可以手动切换“对比基线版本”
- 支持刷新差分结果
- 对比结果按 `sheet -> table -> column` 分层展示

## 差分树与筛选能力

差分树用于快速定位变化点，并支持多维筛选：

- 变化类型：added / removed / modified / renamed
- 字段维度：名称、类型、长度、可空、主键、备注、自增
- 影响分组：breaking / non-breaking / metadata
- 表级模糊搜索（支持拼写容错）

## 重命名确认流程

当系统识别到“可能是重命名而非新增/删除”时，会标记为待确认。

- 可逐项接受或拒绝重命名建议
- 确认结果会参与后续 ALTER 生成
- 可选择是否在导出时包含未确认项

## ALTER 生成与导出

差分确认后可生成 ALTER 预览，支持以下组合：

- 方言：MySQL / Oracle
- 输出模式：single_table / multi_table
- 打包方式：single_file / zip
- 可选项：按 Sheet 拆分、是否包含未确认重命名

结果支持：

- 在线 SQL 高亮预览
- 导出到本地文件（或 ZIP）

## 判定成功信号

出现以下信号可视为差分链路可用：

- 左侧树出现结构化差分节点
- 右侧详情可查看 table/column 级 diff
- ALTER 预览可生成并可导出

若差分树为空或导出失败，请先查看 [故障排查手册](./troubleshooting.md)。

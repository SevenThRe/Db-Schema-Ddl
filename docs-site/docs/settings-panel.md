# 设置面板

设置面板用于定义默认策略。它不改动 Excel 原文件，只影响解析、生成和导出行为。

## 页面总览

![设置页面总览](/img/screens/app-settings-page-full.png)

图例

- 参数分组区：按能力域组织配置
- 参数输入区：文本、下拉、开关、数字输入
- 保存入口：提交当前参数集

## DDL 与 MySQL 规则

这组参数决定 SQL 基础结构和方言细节：

- 注释头、作者名、自定义头模板
- SET NAMES 与 DROP TABLE 开关
- MySQL 引擎、字符集、排序规则
- 数据类型大小写与 BOOLEAN 映射
- VARCHAR 与 CHAR 字符规则

## 导出与解析规则

这组参数决定交付文件和识别边界：

- 文件名前后缀
- 下载路径与 Excel 读取路径
- 连续空行容忍度
- PK 标记列表

## Name Fix 默认策略区

![Name Fix 设置区](/img/screens/settings-name-fix-section.png)

图例

- 默认应用模式
- 冲突策略与保留字策略
- 长度溢出策略与最大标识符长度
- 备份保留天数与并发上限
- Electron 覆盖写入与外部路径写入开关

这个区域会直接影响 Name Fix 工作流的默认行为。

## 开发者与运行守护区

![开发者设置区](/img/screens/settings-developer-section.png)

图例

- 开发者模式开关
- 预热开关
- 上传与解析限流阈值
- 全局防护与在途并发阈值
- 任务队列与失效时限

常规使用建议保持默认值，只在明确性能或稳定性目标时再调节。

## 生效信号

保存后可通过三个信号判断参数是否生效：

- 新生成 SQL 结构与模板输出发生对应变化
- 导出文件命名规则按新参数执行
- Name Fix 弹窗默认策略与设置一致

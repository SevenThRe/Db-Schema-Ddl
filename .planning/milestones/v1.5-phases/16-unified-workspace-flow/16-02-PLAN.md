<objective>
Implement per-connection workspace isolation and reusable SQL memory for Phase 16.

Purpose: A daily DB tool fails when tabs or drafts bleed across environments. This plan introduces one per-connection session store that restores tabs/drafts and includes recent queries plus saved snippets.
Output: Switching connection loads the correct session state; users can reopen recent SQL and save snippets for the active connection.
</objective>

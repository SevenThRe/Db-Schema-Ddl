  <action>
Implement execution phase with table-scoped atomicity and durable job records.

1. Add sqlite tables in storage.rs:
   - db_data_apply_jobs(job_id, compare_id, source_connection_id, target_connection_id, status, action_counts_json, created_at, started_at, finished_at)
   - db_data_apply_results(job_id, table_name, action_type, attempted_rows, succeeded_rows, failed_rows, failure_json)
2. In db_data_apply_execute:
   - create job with status pending then running.
   - for each selected table action batch, open one transaction.
   - commit table transaction on success; rollback entire table batch on first statement failure.
   - continue with remaining tables so overall job can become partial.
3. Compute final job status:
   - completed when all table batches succeed
   - failed when all selected table batches fail
   - partial when mixed success/failure
4. Persist per-table failure diagnostics including key tuple where available.
  </action>

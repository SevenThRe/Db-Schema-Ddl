<objective>
Replace the demo-sized backend query behavior with app-grade runtime semantics.

Purpose: The current backend still loads full result sets into memory before the UI can show the first page, uses a naive LIMIT/OFFSET append path, has no registered export command, and hardcodes PostgreSQL introspection to `public`. This plan removes those blockers in the Rust runtime.
Output: Query execution is page-first, paging support is explicit, export is a real backend command with cancellation semantics, and PostgreSQL schema-aware execution/introspection works beyond `public`.
</objective>

  <name>Task 1: Make query execution page-first and expose fail-closed paging metadata</name>
  <files>src-tauri/src/db_connector/query.rs</files>
  <read_first>
    - src-tauri/src/db_connector/query.rs (current execute_statement, execute_mysql_statement, execute_postgres_statement, db_query_fetch_more)
    - src-tauri/src/db_connector/mod.rs (updated paging fields from Plan 01)
    - src-tauri/src/db_connector/explain.rs (shared driver and SQL-shape conventions)
  </read_first>
  <action>
Rework `src-tauri/src/db_connector/query.rs` so browse-mode queries return a real first page without first materializing the full result set.

1. For single-statement read queries (`SELECT`, `WITH`, `TABLE`, MySQL `SHOW` where safe), build a bounded SQL path that fetches `limit + 1` rows, not the whole result. Use a wrapper form that preserves the original statement text:
   - base SQL = original statement without trailing semicolon
   - paged SQL = `SELECT * FROM (<base SQL>) AS __db_workbench_page LIMIT {limit_plus_one} OFFSET {offset}`

2. Populate query-batch metadata from the bounded fetch:
   - `returnedRows = min(fetched_rows, limit)`
   - `hasMore = fetched_rows > limit`
   - `nextOffset = offset + returnedRows` when `hasMore`
   - `pagingMode = "offset"` only when the statement shape is safe for the wrapper
   - `pagingMode = "unsupported"` and `pagingReason = "Only single result-returning statements support load more."` for unsafe cases such as multi-statement mixes, non-query statements, or query shapes the wrapper cannot preserve safely
   - `totalRows = null` for the page-first path unless an exact count is already known without rerunning the full statement

3. Honor `request.offset` in the initial execute path and in `db_query_fetch_more`; do not append `LIMIT/OFFSET` to the original SQL blindly anymore.

4. Keep the dangerous-SQL and readonly gates exactly as server-side enforcement. The new paging path must not bypass those checks.

5. Remove any code path where MySQL or PostgreSQL query execution calls `.fetch_all(pool)` on the original unbounded SQL and then trims rows in Rust.
  </action>

Implement a real `db_export_rows` runtime command instead of relying on frontend re-execution and client-side Blob generation.

1. In `src-tauri/src/db_connector/commands.rs`, add `db_export_rows` that accepts the new export request from Plan 01 and returns `BinaryCommandResult`.

2. Handle export scopes concretely:
   - `current_page`: serialize exactly the provided current-page rows and columns on the backend
   - `loaded_rows`: serialize exactly the provided accumulated rows and columns on the backend
   - `full_result`: re-run the SQL through the same bounded runtime path in batches, honoring `schema`, until exhaustion or `maxRows`

3. Set an explicit full-export ceiling:
   - define `MAX_EXPORT_ROWS = 100_000`
   - if `scope === "full_result"` and the request does not provide `maxRows`, use `100_000`
   - if the export is truncated at the ceiling, include that fact in the generated filename or summary payload so the UI can warn the operator

4. Use the cancellation registry for export jobs:
   - register a token with a deterministic key such as `export:{request_id}`
   - remove the token on success, error, and cancel
   - make `db_query_cancel` cancel either a query token or an export token when the request ID matches

5. Keep serialization server-side for all export scopes so the runtime surface is truthful: a registered Tauri command exists and owns the heavy path.

<objective>
Finish Phase 23 by making saved connections releasable and by proving the runtime safety controls tell the truth.

Purpose: The current DB Workbench still stores credentials in plaintext metadata and relies on a mix of UI intent and partial runtime guards for safety semantics. This plan moves secrets into secure storage, migrates legacy installs safely, invalidates stale runtime pools after connection changes, and tightens the operator-facing scope/confirmation labels so the UI never over-promises what execute, export, cancel, edit, and sync will actually do.

</objective>

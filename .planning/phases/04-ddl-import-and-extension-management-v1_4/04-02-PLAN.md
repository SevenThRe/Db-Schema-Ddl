**Enable/disable handler (per D-19, D-25, EXTUI-05):**
```typescript
const queryClient = useQueryClient();

async function handleSetEnabled(id: string, enabled: boolean) {
  const { invoke } = await import("@tauri-apps/api/core");
  await invoke<void>("ext_set_enabled", { id, enabled });
  // キャッシュ無効化で sidebar を即時更新（60s staleTime を上書き）
  await queryClient.invalidateQueries({ queryKey: ["extensions", "all"] });
  toast({
    title: enabled ? `${name} 已启用` : `${name} 已禁用`,
    variant: "success",
  });
}
```

For Phase 4, uninstall for db-connector means calling `ext_set_enabled(id, false)` + setting a local `uninstalledIds` state. Show "已卸载" state with muted "重新安装" placeholder button. Use AlertDialog for confirmation with destructive framing (per UI-SPEC copywriting: "卸载后将无法使用，需重新安装才能恢复。是否继续？").

**Extension filtering logic (per D-01, D-02, D-03):**
From `useExtensionHost().extensions`, filter to show ONLY extensions that are conceptually "external". In Phase 4 this means: include `db-connector` (category DbConnector). Exclude `ddl-to-excel`, `excel-to-java-enum`, `excel-to-ts-enum`, `schema-diff` (these are built-in Transformer features).

Simple filter: `extensions.filter(e => e.manifest.id === "db-connector" || e.manifest.kind === "external")`. This covers Phase 4 (only db-connector) and is future-proof for actual external extensions.

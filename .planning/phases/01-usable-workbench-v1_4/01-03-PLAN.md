**SqlEditorPane.tsx** — Create at `client/src/components/extensions/db-workbench/SqlEditorPane.tsx`:

Props:
```typescript
{
  sql: string;
  dialect: DbDriver;
  onSqlChange: (sql: string) => void;
  onExecuteSelection: (sql: string) => void;
  onExecuteScript: (sql: string) => void;
  onExplain: (sql: string) => void;
  onCancel: () => void;
  isExecuting: boolean;
}
```

Implementation:

1. **Monaco Editor** — Use `Editor` from `@monaco-editor/react`:
   - `language="sql"` (Monaco built-in SQL mode — per EDIT-01)
   - `theme` based on current app theme (check existing dark mode pattern)
   - `value={sql}` and `onChange` -> `onSqlChange`
   - `options`: minimap disabled, fontSize 13, fontFamily "JetBrains Mono" (per UI-SPEC), wordWrap "on", scrollBeyondLastLine false, lineNumbers "on"

2. **Keyboard Actions** — In `onMount` callback:
   - `editorInstance.addAction({ id: "db-execute-selection", label: "Execute Selection / Statement", keybindings: [monaco.KeyMod.CtrlCmd | monaco.KeyCode.Enter], run: () => handleExecuteSelection() })` (per EDIT-02)
   - `editorInstance.addAction({ id: "db-execute-script", label: "Execute Full Script", keybindings: [monaco.KeyMod.Shift | monaco.KeyMod.CtrlCmd | monaco.KeyCode.Enter], run: () => handleExecuteScript() })` (per EDIT-03)
   - `editorInstance.addAction({ id: "db-format-sql", label: "Format SQL", keybindings: [monaco.KeyMod.Alt | monaco.KeyMod.Shift | monaco.KeyCode.KeyF], run: () => handleFormatSql() })` (per EDIT-04)
   - `editorInstance.addAction({ id: "db-close-tab", label: "Close Tab", keybindings: [monaco.KeyMod.CtrlCmd | monaco.KeyCode.KeyW], run: () => onCloseTab?.() })`

3. **Statement Block Detection** (per EDIT-02 — when no selection):
   - `getSelectedSqlOrCurrentStatement(editor)` function:
   - If `editor.getSelection()` has non-empty selection text, return that text
   - Otherwise: get full text, get cursor position via `editor.getPosition()`, calculate character offset
   - Split text into statement blocks using quote-aware semicolon split (respect `'`, `"` quotes and `--` line comments)
   - Find which block contains the cursor offset
   - Return that block's text (trimmed)

4. **Format SQL Handler** (per EDIT-04):
   - `import { format } from "sql-formatter";`
   - If selection exists: format only selected text, replace selection
   - If no selection: format entire editor content
   - Dialect mapping: `connection.driver === "mysql" ? "mysql" : "postgresql"`
   - Use `editorInstance.executeEdits` for proper undo support

5. **EXPLAIN auto-detection** (per D-04):
   - In `handleExecuteSelection`: if the SQL starts with `EXPLAIN` (case-insensitive, after trimming), call `onExplain(sql)` instead of `onExecuteSelection(sql)`

6. **Toolbar** — Rendered above the Monaco editor:
   - Height 36px, `bg-panel-muted border-b border-border flex items-center gap-2 px-2`
   - Buttons (shadcn Button, variant "ghost", size "sm"):
     - **Run** (Play icon) — calls handleExecuteSelection, disabled when `isExecuting`. Tooltip: "Run (Ctrl+Enter)"
     - **Explain** (per D-03, `Lightbulb` or `Zap` icon) — calls onExplain with current selection/statement. Tooltip: "Explain"
     - **Format SQL** (AlignLeft icon) — calls handleFormatSql. Tooltip: "Format SQL (Alt+Shift+F)"
     - Separator
     - **Stop** (Square icon) — calls onCancel, only visible when `isExecuting`. Shows spinner during execution, becomes Stop button.
   - All button labels per UI-SPEC Copywriting Contract

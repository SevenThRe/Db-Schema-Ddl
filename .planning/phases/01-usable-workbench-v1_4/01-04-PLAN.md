**WorkbenchLayout.tsx** — Final wiring of all components:

Update the existing WorkbenchLayout to integrate ALL components from Plans 03 and 04:

1. **Results area** (bottom pane of ResizablePanelGroup):
   - Two sub-tabs: "Results" and "Explain" (shadcn Tabs)
   - "Results" tab renders `<ResultGridPane batches={results?.batches ?? []} onLoadMore={handleLoadMore} isLoading={isExecuting} />`
   - "Explain" tab renders `<ExplainPlanPane plan={explainPlan} isLoading={isExplaining} />`
   - `ResultExportMenu` rendered in results tab toolbar area

2. **Dangerous SQL interception** (pre-execution hook):
   - Before executing any SQL, call `hostApi.connections.previewDangerousSql(connectionId, sql)`
   - If `preview.dangers.length > 0`, open `DangerousSqlDialog` with the preview
   - If user confirms (onConfirm), proceed with execution
   - If user cancels (onCancel), abort execution
   - If no dangers detected, execute immediately

3. **State additions**:
   - `results: QueryExecutionResponse | null`
   - `explainPlan: DbExplainPlan | null`
   - `isExplaining: boolean`
   - `dangerPreview: DangerousSqlPreview | null`
   - `showDangerDialog: boolean`
   - `stopOnError: boolean` (default true per D-05)
   - `pendingSql: string | null` (SQL waiting for danger confirmation)

4. **Execution flow**:
   - `handleExecute(sql)`:
     1. Set pendingSql = sql
     2. Call previewDangerousSql
     3. If dangers found -> set dangerPreview, set showDangerDialog = true
     4. If no dangers -> call executeImmediate(sql)
   - `handleDangerConfirm()`:
     1. Close dialog
     2. Call executeImmediate(pendingSql!)
   - `executeImmediate(sql)`:
     1. Set isExecuting = true, generate requestId
     2. Call hostApi.connections.executeQuery({ connectionId, sql, requestId, continueOnError: !stopOnError })
     3. Set results on success, clear on error
     4. Set isExecuting = false

5. **Explain flow**:
   - `handleExplain(sql)`:
     1. Set isExplaining = true
     2. Call hostApi.connections.explainQuery({ connectionId, sql })
     3. Set explainPlan on success
     4. Switch results tab to "Explain"
     5. Set isExplaining = false

6. **Load more flow** (per D-06):
   - `handleLoadMore(batchIndex)`:
     1. Re-execute the batch's SQL with offset = current rows.length, limit = 1000
     2. Append new rows to existing batch
     3. Update status line

7. **Export flow**:
   - `handleExport(format)`:
     1. Get current active batch
     2. Build export content (CSV/JSON/Markdown/SQL Insert) from loaded rows
     3. Trigger browser download

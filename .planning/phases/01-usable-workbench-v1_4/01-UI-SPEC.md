## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, tab inner padding, tree indent |
| sm | 8px | Toolbar button gaps, tab bar padding, list item padding |
| md | 16px | Panel inner padding, toolbar height padding, dialog body |
| lg | 24px | Section dividers, card padding |
| xl | 32px | Major panel gaps (editor ↔ results split) |
| 2xl | 48px | Not used in workbench (reserved for page-level) |
| 3xl | 64px | Not used in workbench |

Exceptions:
- Environment color band: fixed height **28px** (7 spacing units — intentional non-multiple for compact workbench header)
- Monaco editor: no padding override — uses Monaco internal layout
- Result grid row height: **32px** (multiple of 4)
- Column header row height: **28px** (same as env band — visual rhythm)
- Tab bar height: **36px** (9 spacing — matches existing Dashboard tab pattern)

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 13px (text-[13px]) | 400 | 1.5 | IBM Plex Sans | General UI text, status messages, dialog copy |
| Label | 12px (text-xs) | 600 | 1.3 | IBM Plex Sans | Column headers, tab labels, toolbar labels, tree items |
| Heading | 13px (text-[13px]) | 600 | 1.3 | IBM Plex Sans | Section headings, panel titles, dialog titles |
| Code | 13px (text-[13px]) | 400 | 1.6 | JetBrains Mono | SQL editor, inline SQL in confirmation dialog, grid cell values |

Notes:
- 13px is the de-facto compact body for desktop DB tools — matches DataGrip/DBeaver density
- Existing codebase uses text-xs (12px) and text-sm (14px) — 13px bridges these for workbench density
- Grid cell values use JetBrains Mono at 12px (text-xs) for maximum column density
- Timestamps and row-count status lines use text-xs (12px) at weight 400, muted-foreground color

### 60 / 30 / 10 Split

| Role | Distribution | Value |
|------|-------------|-------|
| Dominant (60%) | Background, editor chrome, results area | `bg-background` |
| Secondary (30%) | Left sidebar, tab bar, toolbar, grid header row | `bg-panel-muted` / `bg-sidebar` |
| Accent (10%) | Active tab underline, Run button, focus ring, selected row highlight | `bg-primary` / `text-primary` |

Accent reserved for:
1. Active query tab underline indicator (2px bottom border, `bg-primary`)
2. "Run" button primary variant
3. Selected row in result grid (`bg-primary/10`)
4. Monaco editor cursor and selection highlight (Monaco-internal, not overridden)
5. Focus ring on interactive elements (`ring-primary`)

### Phase 1 Semantic Overlays (new — not in existing tokens)

These are environment band colors — **add to index.css** as new CSS variables:

```css
/* DB 工作台 — 環境帯カラー */
--env-prod: 0 84% 60%;          /* red — same hue as --destructive */
--env-test: 217 74% 54%;        /* blue — same as --statusbar-accent */
--env-dev: 142 71% 45%;         /* green — same as status.online */
--env-prod-fg: 0 0% 100%;
--env-test-fg: 0 0% 100%;
--env-dev-fg: 0 0% 100%;
```

Usage rules:
- Environment band is a **full-width strip** pinned to the top of the workbench panel, always visible
- Height: 28px, font: 12px (text-xs), weight 600, text center-aligned
- prod: `bg-[hsl(var(--env-prod))] text-[hsl(var(--env-dev-fg))]` + lock icon if readonly
- test: `bg-[hsl(var(--env-test))] text-[hsl(var(--env-test-fg))]`
- dev: `bg-[hsl(var(--env-dev))] text-[hsl(var(--env-dev-fg))]`
- No environment assigned: no band rendered (transparent, no height reserved)

## Layout Contract

**Primary focal point:** The Monaco SQL editor is the dominant visual anchor — it occupies the largest area (flex-1), uses `bg-background` (full contrast), and is framed by secondary-surface toolbars above and a resizable results area below. The eye enters at the editor and flows down to results.

│  [Environment Band — 28px — full width, conditional]     │

│  (fixed)    │  │ [Tab 1] [Tab 2] [+]     36px          │ │

## Interaction Contract

### Keyboard Shortcuts (Monaco addAction)

| Action | Binding | Context |
|--------|---------|---------|
| Run selection / current statement | Ctrl+Enter (Win/Linux), Cmd+Enter (Mac) | Editor focused |
| Run full script | Shift+Ctrl+Enter / Shift+Cmd+Enter | Editor focused |
| Format SQL | Alt+Shift+F | Editor focused |
| Close current tab | Ctrl+W | Editor focused |

### Result Grid Behavior

- Column headers: sticky, `bg-panel-muted`, text-xs weight 600
- Row height: 32px fixed (react-window FixedSizeList)
- Column width: draggable handle, minimum 60px, default auto-fit
- Selected row: `bg-primary/10`
- Null values: render as `null` in text-xs muted-foreground italic
- Long text cells: truncated with ellipsis; full value on hover tooltip

### EXPLAIN Graph Behavior

- Auto-layout via elkjs `layered` algorithm, direction `RIGHT`
- Full-table-scan nodes: `border-2 border-[hsl(var(--explain-risk-high))]` + red background tint
- Large-rows nodes: amber `Badge` with LARGE ROWS label
- Normal nodes: `border border-border bg-card`
- Node label: table name (text-xs, weight 600) + operation type (text-xs, muted)
- Edge: default xyflow edge, `stroke: hsl(var(--border))`
